# DataForge v1.0 MVP 设计文档

**日期:** 2026-03-06
**状态:** 已批准
**作者:** AI 辅助设计

---

## 背景与目标

v0.5 Beta 已完成所有核心引擎组件（异步调度、限流、断点续传、EvolInstruct、RegexFilter、LLMJudge）。

v1.0 MVP 的唯一新增功能是：**声明式 YAML 配置文件 + `dataforge run config.yaml` CLI 入口**，实现对算法工程师和数据产品经理的零代码使用体验，同时完成正式开源发布前的收尾工作。

---

## 设计原则

1. **零侵入**：现有任何模块不修改，只做增量添加。
2. **快速失败**：配置错误在解析阶段（运行前）即报出字段级提示，不等到运行时。
3. **无新重依赖**：Pydantic 已是现有依赖，只新增 `pyyaml>=6.0`。

## 配置 Schema 设计

### YAML 格式（完整示例）

```yaml
name: "medical-reasoning-dataset"

source:
  type: "jsonl"
  path: "./data/medical_seeds.jsonl"

pipeline:
  - step: "generate"
    strategy: "evol-instruct"
    depth: 2
    mutation_types: ["constraints", "deepen"]
    require_reasoning: false
    require_json: false
    llm:
      provider: "vllm"
      model: "Qwen/Qwen2.5-7B-Instruct"
      base_url: "http://localhost:8000/v1"
      concurrency: 50
      rpm_limit: 1000

  - step: "evaluate"
    evaluator: "regex-filter"
    blacklist_patterns: ["抱歉", "我无法"]

  - step: "evaluate"
    evaluator: "llm-judge"
    criteria: "helpfulness"
    threshold: 4.0
    llm:
      provider: "openai"
      model: "gpt-4o-mini"
      rpm_limit: 500
      tpm_limit: 200000

sink:
  path: "./data/output/high_quality_medical.jsonl"
  checkpoint_dir: "./checkpoints/exp_v1"
```

### Pydantic 模型层次

```
ForgeConfig
├── name: str
├── source: SourceConfig
│   ├── type: Literal["jsonl"]
│   └── path: str
├── pipeline: list[GenerateStepConfig | EvaluateStepConfig]
│   ├── GenerateStepConfig
│   │   ├── step: Literal["generate"]
│   │   ├── strategy: Literal["evol-instruct"]
│   │   ├── depth: int = 3
│   │   ├── mutation_types: list[str] | None = None
│   │   ├── require_reasoning: bool = False
│   │   ├── require_json: bool = False
│   │   └── llm: LLMConfig
│   └── EvaluateStepConfig
│       ├── step: Literal["evaluate"]
│       ├── evaluator: Literal["llm-judge", "regex-filter"]
│       ├── llm: LLMConfig | None = None       # llm-judge 需要
│       ├── criteria: str = "helpfulness"
│       ├── threshold: float = 4.0
│       ├── blacklist_patterns: list[str] = []
│       └── require_json: bool = False
└── sink: SinkConfig
    ├── path: str
    └── checkpoint_dir: str = "./.dataforge_runs"

LLMConfig
├── provider: Literal["openai", "vllm", "anthropic"]
├── model: str
├── base_url: str | None = None
├── api_key: str | None = None   # 未填则从环境变量自动读取
├── rpm_limit: int = 60
├── tpm_limit: int = 100_000
├── concurrency: int = 50
└── generation_kwargs: dict = {}
```

### 环境变量自动读取规则

| provider    | 自动查找的环境变量   |
|-------------|---------------------|
| `openai`    | `OPENAI_API_KEY`    |
| `anthropic` | `ANTHROPIC_API_KEY` |
| `vllm`      | 不需要 key           |

## Loader 设计

```python
# config/loader.py

def load_config(path: str) -> ForgeConfig:
    """读取 YAML 文件 → Pydantic 校验 → 返回 ForgeConfig"""

def build_client(cfg: LLMConfig) -> BaseLLMClient:
    """根据 provider 实例化对应 LLM 客户端"""

def build_strategy(step: GenerateStepConfig) -> BaseStrategy:
    """根据 strategy 字段实例化对应策略"""

def build_pipeline(config: ForgeConfig) -> tuple[Pipeline, str, str, int]:
    """组装 Pipeline，返回 (pipeline, input_path, output_path, concurrency)"""
```

pipeline 中多个 `evaluate` step 会按顺序收集为 `evaluators` 列表传给 `Pipeline`。
`concurrency` 来自 `generate` step 的 `llm.concurrency`。

## CLI 设计

```
dataforge <command> [args]

Commands:
  run <config.yaml>   根据 YAML 配置运行数据处理管线
      --dry-run       仅验证配置，不执行
```

实现：`src/dataforge/cli.py` 中的 `main()` 函数，使用标准库 `argparse`（无额外依赖）。

注册：
```toml
[project.scripts]
dataforge = "dataforge.cli:main"
```
